# PRD — Byredo Interaction Commerce Website (MVP)

## 1. 서비스 목적
이 서비스는 **바이레도의 브랜드 감도와 세계관을 웹 상에서 경험**할 수 있도록 하기 위해 만든다.  
단순한 상품 나열이 아닌, **인터랙션 중심의 제품 탐색 경험**을 제공한다.

## 2. 타겟 사용자
- 바이레도 브랜드를 이미 알고 있거나, **감도 높은 브랜드 경험을 선호하는 20–40대 사용자**

## 3. 범위 (MVP)
- 1차 버전은 **브랜드 경험 + 최소 커머스 액션**에 집중한다.
- 실제 결제/회원 등은 제외하고, “탐색 → 상세 경험 → 장바구니/찜” 흐름을 완성한다.

## 4. 핵심 기능 (3개)
### F1. 브랜드 중심 제품 탐색
- 스크롤 기반 인터랙션으로 제품을 자연스럽게 노출
- 컬렉션 단위로 제품 탐색 가능

### F2. 3D / 비주얼 중심 제품 상세 보기
- 제품을 단순 이미지가 아닌 **3D 오브젝트**로 경험
- 제품 정보는 최소 텍스트로 제공(비주얼 우선)

### F3. 기본 커머스 액션
- 장바구니 담기
- 관심 상품(찜) 등록

## 5. IA / 페이지 목록
- Home (메인)
- Products (상품 리스트)
- Product Detail (상품 상세, 3D)

## 6. 페이지별 요구사항 (UI 요소 포함)

### 6.1 Home (메인 페이지)
**목표**
- 첫 진입에서 브랜드 무드 전달 + 탐색 진입점 제공

**구성/섹션**
1) 하프앤하프 캐러셀 비주얼 섹션  
2) 상품 썸네일 4×4 그리드  
3) 뉴스레터 섹션  
4) CTA 버튼 영역

**필수 UI 요소**
- 하프앤하프 캐러셀
  - 좌/우(또는 상/하) 분할 레이아웃 컨테이너
  - 캐러셀 네비게이션: `이전/다음` 버튼 또는 인디케이터(도트)
  - 브랜드 키 메시지 텍스트(타이틀/서브)
  - 스크롤 유도 UI(아이콘/텍스트)
- 상품 썸네일 4×4
  - 상품 카드(직사각형 썸네일)
  - 상품명 텍스트(최소)
  - 카드 전체 클릭 영역(상세 이동)
- 뉴스레터
  - 이메일 입력창
  - `구독하기` 버튼
  - 유효성 안내 문구(에러/성공)
- CTA 버튼
  - `제품 보러가기` 버튼(Products 이동)
  - `컬렉션 보기` 버튼(동일 페이지 내 앵커 or Products 상단 필터로 연결)

**상태/예외**
- 캐러셀 로딩 중: 스켈레톤/플레이스홀더
- 뉴스레터 이메일 형식 오류: 인풋 하단 에러 텍스트

---

### 6.2 Products (상품 페이지)
**목표**
- 제품을 빠르게 탐색하고 상세로 진입

**구성/섹션**
1) 상품 썸네일 4×4 그리드(페이지네이션 또는 무한스크롤은 선택)  
2) (선택) 컬렉션 구분/정렬 UI

**필수 UI 요소**
- 4×4 그리드 레이아웃
  - 상품 카드(직사각형 썸네일)
  - 상품명 텍스트
  - 카드 클릭 → Product Detail 이동
- (선택) 컬렉션/정렬 UI
  - 컬렉션 탭(텍스트 버튼) 또는 드롭다운
  - 정렬 드롭다운(예: 인기순/신상품순) — MVP에서는 없어도 됨

**상태/예외**
- 이미지 로딩 실패: 대체 이미지/아이콘
- 데이터 없음: “상품이 없습니다” 안내 문구

---

### 6.3 Product Detail (상품 상세 페이지 — 3D)
**목표**
- 3D 인터랙션으로 제품을 경험하고 장바구니/찜 액션 수행

**구성/섹션**
1) 3D 인터랙션 뷰어(메인)  
2) 제품 정보(최소 텍스트)  
3) 액션 버튼(구매/찜)  
4) 보조 정보(향 노트/브랜드 스토리 일부)

**필수 UI 요소**
- 3D 뷰어 영역
  - 로딩 인디케이터(3D 로딩 중)
  - 드래그/스크롤 인터랙션 힌트(“Drag to rotate” 등)
  - (선택) 확대/축소 컨트롤(+/-) 또는 제스처 안내
- 제품 정보 영역
  - 제품명
  - 간단한 향 설명(짧게)
  - 용량 / 가격
- 액션 버튼
  - `장바구니 담기` 버튼
  - `찜하기` 토글 버튼(활성/비활성 상태)
- 보조 정보
  - 향 노트 리스트(텍스트)
  - 브랜드 스토리 일부(짧은 문단)

**상태/예외**
- 3D 로딩 실패: 정적 이미지 대체 + 안내 문구
- 장바구니 추가 성공: 토스트/스낵바 피드백
- 찜 토글: 즉시 상태 반영(아이콘/텍스트 변화)

## 7. 사용자 흐름 (User Flow)
1. Home 진입  
2. 캐러셀/스크롤로 브랜드 경험  
3. 썸네일 클릭 또는 CTA 클릭  
4. Products 진입 → 4×4 그리드 탐색  
5. 특정 상품 클릭  
6. Product Detail 진입 → 3D 인터랙션으로 확인  
7. `장바구니 담기` 또는 `찜하기` 수행  
8. (MVP 종료) — 결제/로그인은 향후 기능

## 8. 비기능 요구사항 (간단)
- 모바일/데스크톱 반응형 필수
- 3D 로딩이 길어질 경우를 대비해 **fallback 이미지** 제공
- 스크롤/호버 인터랙션은 과하지 않게(성능/가독성 우선)

## 9. 향후 추가 기능 (1차 제외)
- 회원가입 / 로그인
- 실제 결제 시스템 연동
- 리뷰 및 평점
- 개인 맞춤 향 추천
- 관리자 페이지(상품/재고/콘텐츠 관리)
- 검색 기능
- 페이지네이션/무한스크롤 고도화

## 10. MVP 완료 기준 (Acceptance)
- 사용자가 **Home → Products → Product Detail** 흐름으로 1분 내 탐색 가능
- Product Detail에서 **3D 경험 + 장바구니/찜** 액션이 정상 동작
- 3D 실패 시에도 정적 이미지로 제품 확인 가능
